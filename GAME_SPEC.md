# 《点灯人》Game Spec v0.1

> 一款关于劳作、记忆与城市更替的 2D 叙事小游戏，同时用于验证“AI 生成带动画的 2D 角色资产”的完整工作流。

## 1. 一句话概念

旧城的煤气路灯将在天亮后永久停用。玩家扮演最后一位点灯人，提着灯、背着梯子，走完他最后一次巡夜。

## 2. 体验目标

- 玩家感到的不是“拯救世界”，而是一个普通人对一份旧工作的认真告别。
- 人物的年龄、习惯、疲惫和尊严，主要通过动作而非台词呈现。
- 每点亮一盏灯，街区便短暂恢复一小块生活痕迹。

## 3. 主角：周师傅

- 58–65 岁，孤寡、寡言，不英雄化。
- 穿修补过的深蓝工装、旧皮鞋，戴软檐帽。
- 长期背梯导致轻微驼背，走路不快，但操作灯具时稳定、熟练。
- 会下意识擦净灯罩，在离开前回头确认火苗。
- 不使用夸张表情；人物性格由肩背、步态、停顿和手势表达。

## 4. 核心玩法

### 基本循环

```text
沿旧街行走
→ 找到熄灭的路灯
→ 放下梯子
→ 检查故障
→ 清理 / 换灯芯 / 补充灯油
→ 点灯并护住火苗
→ 看见这盏灯下的一段人间记忆
```

### 操作

- 左右移动。
- 短按互动：拾取、检查、放置。
- 长按互动：擦拭、修理、点灯。
- 风吹来时按住“护灯”，调整身体方向挡风。

### 轻度挑战

- 不设战斗。
- 挑战来自风、雨、不同故障和天亮前的有限时间。
- 失败不立即结束游戏；火苗被吹灭只需重新操作，避免破坏安静节奏。

## 5. 首个可玩版本（Vertical Slice）

- 一条可横向行走的旧街，总流程 8–12 分钟。
- 3 盏路灯，分别对应三种交互：
  1. 擦净被煤灰覆盖的灯罩。
  2. 更换断掉的灯芯。
  3. 在雨中点灯并护住火苗。
- 3 段无配音的短记忆：等待收工的小吃摊夫妻、夜读的学生、清晨扫街的环卫工。
- 结尾：天色发白，电灯同时亮起，周师傅主动吹灭手中的灯。

## 6. 美术方向

### 风格

**石版画的颗粒 × 旧绘本的人物 × 克制的手绘动画**

- 非 Q 版，避免通用“治愈系可爱小人”。
- 人物轮廓保留手绘的不规则，但脚底线和工具接触点必须稳定。
- 画面以大面积暗色和局部灯光组织，不使用多色霓虹。
- 动画建议 12 FPS，保留少量手绘沛动，不追求机械平滑。

### 限定色板

| 用途 | 颜色 |
|---|---|
| 夜色 / 工装 | 深靛蓝 |
| 墙面 / 旧纸 | 灰褐与暖灰 |
| 铁锈 / 记忆重点 | 暗砖红 |
| 灯火 / 生活痕迹 | 暖琥珀金 |

## 7. 声音方向

- 不使用持续铺满的情绪音乐。
- 主要声音：鞋底、梯子碰地、灯罩擦拭、火柴、风、远处电车和清晨鸟声。
- 每盏灯点亮后，增加一层该地点的生活声，让整条街逐渐“被听见”。

## 8. AI 资产生成验证矩阵

### 角色动作集

| 类别 | 动作 | 主要验证问题 |
|---|---|---|
| 基础 | 待机、行走、转身 | 五官、年龄、服装、步态一致性 |
| 负重 | 背梯行走、放下梯子、扛起梯子 | 人与长物体接触、遮挡和重心 |
| 精细劳作 | 擦灯罩、换灯芯、加灯油 | 手部、小工具与定制动作理解 |
| 火焰互动 | 划火柴、点灯、侧身护灯 | 动态光影、手眼对齐与火焰特效 |
| 人物性 | 擦汗、捶腰、短暂休息、回头看灯 | 年龄感、克制表演与角色性格 |
| 环境反应 | 挡风、雨中滑步、火苗熄灭后的停顿 | 物理感、状态切换和失败表演 |

### 必须故意保留的资产缺口

1. 首次只生成“待机 + 行走 + 点灯”，开发中途追加“换灯芯”，验证新动作能否无损加入。
2. 先不预统一画布和脚底线，记录导入 Cocos 后的真实返工量。
3. 至少一次从生成视频抽帧，观察背景分离、帧间漂移和节奏问题。
4. 在人物已定稿后追加“梯子”这个长道具，验证角色与道具关系。
5. 保留一个失败动作并尝试“局部重生成”，验证是否必须整套推倒重来。

## 9. 资产交付标准（首版）

- 格式：透明 PNG 序列帧 + Sprite Sheet + 描述文件。
- 基准尺寸：角色可见高度约 320 px，实际画布预留梯子与动作外延。
- 帧率：默认 12 FPS，每个动作允许独立配置。
- 锨点：角色脚底中点；交互动作另记录手部接触点。
- 命名：`lamplighter_<action>_<index>.png`。
- 描述数据：动作名、帧顺序、FPS、循环方式、脚底锨点、手部接触点、生成模型、提示词版本。

## 10. Cocos 技术范围

- 引擎：Cocos Creator 3.8.8。
- 类型：2D 横向叙事探索。
- 脚本：TypeScript。
- 首版平台：Web 预览；第二验收平台为微信小游戏。
- 必要系统：角色移动、交互状态机、动画播放、路灯状态、天色计时、音频分层、一页开始/结束界面。
- AI API 仅由本地资产工具调用，API Key 不进入 Cocos 客户端、Git 或 Obsidian。

## 11. 验证记录

每组资产记录：

- 需求描述与提示词版本。
- 模型、参数、生成时长与费用。
- 候选数量与最终采用率。
- 手工操作步骤、点击次数与耗时。
- 帧间漂移、尺寸、脚底线、透明度、道具接触等失败类型。
- 导入 Cocos 后新暴露的问题。
- 返工是局部修复还是整套重做。

## 12. 首版不做

- 不做战斗、成长、商店、装备和地图分支。
- 不做多角色可玩。
- 不做大量对话和全程配音。
- 不在游戏内接入生成 API。
- 不先做通用资产平台，先完成一次真实生产。

## 13. 第一阶段验收

- 玩家可从街头走到街尾并点亮 3 盏灯。
- 至少 8 组主角动作真正进入实机玩法，不只停留在预览 GIF。
- 完成一次“已有角色追加新动作”。
- 完成一次从生成视频到 Sprite Sheet 再到 Cocos 动画的闭环。
- 记录全流程耗时、费用、人工步骤与失败点。
- Web 版可独立运行，无阻断性错误。

## 14. 待定决策

1. 周师傅的具体地域与年代特征。
2. 是否保留少量文字旁白，还是完全无字叙事。
3. 生成动画的首选路线：参考图逐帧、图生视频抽帧，或两者混合。
4. 微信小游戏是否在 Vertical Slice 阶段就立即构建验收。
