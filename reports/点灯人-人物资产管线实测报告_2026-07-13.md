# 点灯人 · 人物资产管线实测报告

## 1. 结论先行

本轮已经跑通“角色母版 → 多视角 / 多动作生成 → 去背 → 切帧 → 统一画布与脚底线 → 自动质检 → 人工逐帧审核 → metadata 导出”的最小闭环，并完成独立人物资产实验室。

结果支持 MS1 的核心判断：真正的产品价值不在“能否生成一张好看的角色图”，而在于能否把模型输出稳定地变成可追溯、可审核、可局部返工、可直接进入 Cocos 的动作资产。

当前工程后处理已经能稳定解决透明背景、画布、命名、脚底线和播放问题；尚未解决的主要风险是跨视角身份一致性、复杂动作的逐帧连续性，以及视觉缺陷无法仅凭几何质检发现。

## 2. 本轮范围

按 MS1 的资产模型组织：

```text
项目：点灯人 MVP
└─ 角色身份：点灯少年（已锁定）
   └─ 造型：常态造型 v1
      ├─ 横屏侧视 / walk（8 帧）
      ├─ 横屏侧视 / run（8 帧）
      ├─ 横屏侧视 / lantern interaction（8 帧）
      ├─ 俯视 / walk（8 帧）
      └─ 2.5D / walk（8 帧）
```

另建立 5 视角角色母版：侧面、正面、背面、高位 3/4、2.5D 3/4。后续动作不得独立抽卡，必须引用该母版与当前造型基准帧。

## 3. 交付物

### 3.1 人物资产实验室

独立入口：`http://127.0.0.1:4174/asset-lab/`

已实现：

- 项目 → 角色身份 → 造型 → 动作实例的资产树。
- 横屏、俯视、2.5D 三套真实资产切换；缺资产时明确显示缺口，不做伪透视。
- 动画播放 / 暂停、逐帧、FPS、循环控制；默认 16 FPS，8 FPS 作为最低慢放审核档。
- 帧级状态机：待审核 → 通过 / 退回；逐帧操作自动暂停。
- 自动质检：画布、透明背景、脚底线偏差、主体高度偏差。
- 生成批次追溯：批次、模型、路线、帧来源。
- 导出门禁：只有动作全部帧通过才允许导出 metadata。
- 审核状态写入浏览器本地存储，刷新后保留。

## 4. MS1 三个数

### 4.1 生成次数

- 当前工作区共保留 10 张角色相关原始生成图，另有 1 张场景图。
- 本轮生产候选链路使用 6 次关键角色生成：横屏 walk、5 视角母版、俯视 walk、2.5D walk、横屏 run、横屏举灯。
- 早期 4 次人物方向探索（老人、工装少年、写实青年、Q 版青年）均保留，用于记录角色定调成本。

### 4.2 耗时

- 本轮生产候选资产从横屏 walk 落盘到扩展动作落盘，观测墙钟时间约 20 分钟（15:18–15:38），包含母版生成、多视角 / 动作生成、去背与切帧。
- Cocos Web 构建约 3–11 秒 / 次；后续增量构建约 3 秒。
- 当前没有把“单次模型纯耗时”自动写入批次，这是必须补齐的埋点。

### 4.3 成本

- 当前生成工具未返回可核算费用，实际成本无法可靠报告。
- 结论：成本字段已经进入批次模型与界面，但数据采集仍是缺口；正式实验必须记录请求级模型、尺寸、质量、计费量与金额。

## 5. 关键问题与产品启示

### 5.1 一张 Sprite Sheet 不是一个可用动作

早期图虽然包含多个人物姿势，但并非严格步态。直接播放会出现同一条腿领先、角色忽大忽小、鞋尖反向等问题。仅提高 FPS 会更快地暴露抖动，不会产生缺失的中间动作。

产品启示：动作定义必须显式包含接触、下压、经过、上升等相位；生成后按动作语义检查，不能只检查“有 8 张图”。

### 5.2 不应从成图中强行镜像局部肢体

尝试把下半身镜像以制造另一迈步相位，导致鞋尖、衣摆和关节反向；尝试切腿换位又产生断膝。该路线已判定不适合产品化。

产品启示：坚持 MS1 的参考图 + 逐帧生成路线；坏帧单独重生成，并携带相邻帧上下文，不走自动骨骼或图片切件路线。

### 5.3 多视角必须是不同资产

横屏、俯视和 2.5D 无法通过 CSS 旋转 / 压缩互相替代。当前已改为先锁定 5 视角母版，再分别生成动作实例。

产品启示：方向应进入动作定义规格；`walk / side`、`walk / topdown`、`walk / isometric` 是共享动作定义的三个实例，而不是同一 PNG 的显示模式。

### 5.4 母版能改善一致性，但不能保证一致性

统一母版后，发型、围巾、海军蓝外套和黄铜配件明显更稳定；但俯视与 2.5D 的镜头差异仍不够大，部分帧的脸型、衣摆长度仍有概率漂移。

产品启示：角色卡必须从“提示词附件”升级为可量化约束，至少记录头高、发型轮廓、主色值、服装分区和关键配饰位置。

### 5.5 工程质检不能替代视觉审核

所有 5 套动作的画布、透明背景、脚底线和主体高度检查均为 4/4 通过，但举灯动作仍出现视觉边缘残留。

产品启示：自动质检负责筛掉工程错误；人确认负责角色一致性、解剖、动作可读性和特效边缘。两者必须串联。

### 5.6 生成尺寸不能假设可整除

模型生成的 8 格图宽度可能不是 8 的整数倍。固定宽度切片会逐格累计偏移。

已采用按比例计算每格左右边界的切帧方式；长期方案应要求模型输出帧坐标或先检测角色中心，而不是依赖理想网格。

## 6. 验证记录

- 人物资产实验室页面加载成功，无浏览器 error / warning。
- 横屏 walk、run、举灯，俯视 walk、2.5D walk 均能播放。
- 五套动作自动质检均为 4/4 通过。
- 视角切换加载不同目录下的真实帧资产。
- 缺失动作显示“资产缺口”，不会用其他视角伪装。
- 逐帧点击、退回、通过、状态进度与导出门禁工作正常。
- 审核动作会自动暂停，避免播放过程中误审下一帧。
- 横屏 walk 在 Cocos Creator Web 构建中首次播放成功。
- 新增侧视 idle / jump、俯视 run、2.5D run，共新增 4 组、32 张透明帧。
- 帧率已在平台播放、metadata 导出、联调协议与 Cocos 运行时统一锁定为 8 FPS。
- 自动质检增加相邻帧质心位移尖峰、轮廓面积波动、循环首尾接缝三项几何连续性指标。
- 横屏、俯视、2.5D 切换已加入淡出、轻微缩放与清晰度过渡，不再硬切。
- 资产实验室可将当前视角、动作、循环设置推送到 Cocos；实测 `topdown / walk · 8帧` 返回应用成功。
- “进入 Cocos 游戏”按钮可一键打开独立游戏页并带入当前审核动作。
- 最终自动化实测：FPS=8 且不可修改；视角过渡类生效；Cocos 同步成功；独立游戏页打开；无脚本错误。
- 预览舞台新增直接操控与自动巡走：支持 A / D、左右方向键及按钮长按；移动时自动切入 walk，停止时在有 idle 资产的视角自动切回 idle，左右边界自动转身。实测手动移动、停止待机和自动巡走均通过，无脚本错误。

## 7. 下一轮优先级

1. 把提示词、参考图版本、模型、开始 / 结束时间、请求成本自动写入批次。
2. 为角色卡增加可计算特征：头高、轮廓、调色板、服装区域与配饰锚点。
3. 对视觉审核增加“脸 / 发型 / 服装 / 配色 / 体型 / 解剖 / 动作相位”七项清单。
4. 打通单帧退回 → 带相邻帧上下文重生成 → 新批次 → 原位置替换。
5. 继续补齐俯视与 2.5D idle / jump，横屏再补 attack；当前横屏 idle / jump 及俯视、2.5D run 已完成。
6. 生成真正的 Cocos 导出目录：透明 PNG、sprite sheet、metadata、预览 GIF 与导入说明。

## 8. 最终判断

这次实测不是证明“AI 已经能稳定生成完整游戏角色动画”，而是更清楚地证明：AI 输出仍需要角色卡约束、动作规格、确定性后处理、自动质检、逐帧审核、批次追溯和导出门禁共同组成产品。

因此，Windup 当前的正确方向仍是：**AI 生成，人确认，工程系统保证交付契约。**
